简介

X-Lily6x（简称 Lily）是一个 X-Lily6x 的增强工具，在 Mybatis 的基础上只做增强不做改变，为简化开发、提高效率而生。

我们的愿景是成为X-Lily6x最好的搭档，就像 Contra Game 中的1P、2P，基友搭配，效率翻倍。

特性

  • 无侵入：X-Lily6x 在 X-Lily6x 的基础上进行扩展，只做增强不做改变，引入 X-Lily6x 不会对您现有的 X-Lily6x 构架产生任何影响，而且 MP 支持所有 Mybatis 原生的特性
  • 依赖少：仅仅依赖 Mybatis 以及 Mybatis-Spring
  • 损耗小：启动即会自动注入基本CURD，性能基本无损耗，直接面向对象操作
  • 预防Sql注入：内置Sql注入剥离器，有效预防Sql注入攻击
  • 通用CRUD操作：内置通用 Mapper、通用 Service，仅仅通过少量配置即可实现单表大部分 CRUD 操作，更有强大的条件构造器，满足各类使用需求
  • 多种主键策略：支持多达4种主键策略（内含分布式唯一ID生成器），可自由配置，完美解决主键问题
  • 支持热加载：Mapper 对应的 XML 支持热加载，对于简单的 CRUD 操作，甚至可以无 XML 启动
  • 支持ActiveRecord：支持 ActiveRecord 形式调用，实体类只需继承 Model 类即可实现基本 CRUD 操作
  • 支持代码生成：采用代码或者 Maven 插件可快速生成 Mapper 、 Model 、 Service 、 Controller 层代码，支持模板引擎，更有超多自定义配置等您来使用（P.S. 比 Mybatis 官方的 Generator 更加强大！）
  • 支持自定义全局通用操作：支持全局通用方法注入（ Write once, use anywhere ）
  • 支持关键词自动转义：支持数据库关键词（order、key......）自动转义，还可自定义关键词
  • 内置分页插件：基于Mybatis物理分页，开发者无需关心具体操作，配置好插件之后，写分页等同于普通List查询
  • 内置性能分析插件：可输出Sql语句以及其执行时间，建议开发测试时启用该功能，能有效解决慢查询
  • 内置全局拦截插件：提供全表 delete 、 update 操作智能分析阻断，预防误操作

快速入门

添加 X-Lily6x 支持

X-Lily6x 扩展于 jQuery ， 所以在引入js 时，需要先引用 jQuery 版本在1.8以上都可以

引入 X-Lily6x js文件


    <script src="../resources/js/jquery-2.1.1.js" ></script>
    <script src="../resources/js/jquery-xlily6x-ui-1.0.0.js" ></script>

引入css样式


    <link rel="stylesheet" type="text/css" href="../resources/css/jquery-xlily6x-ui-1.0.1.css"/>

使用 X-Lily6x 控件

X-Lily6x 的所有控件使用方式都一样，通过jQuery 对象即可直接调用


    $("#const").xLily6X("widgetName",options);

xLily6X("widgetName",options) 参数说明

名称 类型 默认值 描述
widgetName string "" 在 X-Lily6x 的世界中，所有的控件都有一个名字,这个参数表示需要加载的控件名 控件名可通过 $.widget 对象获取
options object 加载控件所需要的一些初始化配置信息,不同的控件有不同options，具体控件options请看对应控件参数说明

右键菜单

控件说明

使用该控件后，当鼠标移至 Target 上时，右键单击，将出现一个关联菜单，用户可根据自己需求设置菜单中的菜单 项，以及单击后的处理事件。

Html 代码

界面有div元素，指定 id值


    <div id="const" style="width: 500px;height: 300px; background-color: #b9c7c2">
        <label id="test1">test11111</label><br/>
        <label id="test2">test222222</label><br/>

    </div>

通过jQuery对象即可调用菜单空间方法

            
    // 初始化 右键环境菜单
    $("#const").xLily6X($.widget.ContextMenu,options);

options 对象示例

            
    /**
     * 环境菜单 选项
     */
    var options = {
        // 菜单项
        items:[
            {id:"saveBtn",text:"save0",click:function (obj,env) {
                    console.log(obj);
                    console.log(env);
                }
            },
            {id:"save2Btn",text:"save1",click:
                // 单机事件
                function (obj,env) {console.log(obj);console.log(env);}},
            {id:"save3Btn",text:"save2",click:function (obj,env) {console.log(obj);console.log(env);}}
        ],
        auto_height:true,
        //背景颜色
        background_color:"#fbf8ff"
    }

options 参数说明

每一个对象都会有一个默认原型，当缺省时，会根据原型中的默认值进行默认加载

名称 类型 默认值 描述
items object "" items 参数用来指定菜单项属性，指定菜单项的ID ，显示的文本，和单机时触发的事件等等，具体可参见下面 items 参数说明
width int 100 菜单的整体宽度，单位 px
height int 100 菜单的整体高度，单位 px ,如需指定高度，请将auto_height 设为false
auto_height boolean true 是否自动设置高度，为true时，设置 height 值无效
background_color string #fbf8ff 背景颜色 默认米白色，如需指定颜色，background_color 设为false
background_color boolean false 自定义设置背景颜色，默认为 false

菜单 options 对象原型


    /*
     *  菜单Options 原型 默认值
     */
    MenuOptions:{
        // 宽度 单位 px
        width:100,
        // 高度 单位 px
        height: 100,
        // 是否自动设置高度，为true时，设置 height 值无效
        auto_height:true,
        // 背景颜色
        background_color:"#fbf8ff",
        // 是否自定义背景颜色，为true时，backgroud_color 设置 无效
        custom_background_color:false,
        items:[
            {id:"menu_item_edit",text:"edit this",click:function (obj,env) {
                console.log(obj);
            }},
            {id:"menu_item_delete",text:"delete this",click:function (obj,env) {
                console.log(obj);
            }}
        ]
    },

item 参数说明

名称 类型 默认值 描述
id string "" 菜单按钮的ID值，可通过这个ID值获取到按钮对象，
text string "" 菜单按钮显示的文本信息
click function 菜单按钮的单机处理事件，如需更多事件，可根据ID获取到对象后自行扩展
